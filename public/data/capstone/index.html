Abstract

This project implements a chatbot-like application powered by a Retrieval-Augmented Generation (RAG) model to generate accurate, context-aware responses to user queries.

Traditional RAG systems use vector search for retrieval but often struggle with unstructured data. Our approach integrates knowledge graphs, adding structured relationships to improve retrieval accuracy and response quality. The system processes 10,000 Wikipedia articles along with additional datasets, using enriched vector embeddings to retrieve relevant information. It follows a client-server architecture, with a React frontend for interaction and a backend handling vector search, knowledge graph processing, and LLM-based response generation.

Key challenges include refining query handling for complex relationships and optimizing response time. Future improvements will focus on enhancing robustness, expanding multimodal capabilities, and refining retrieval techniques for more advanced NLP applications.

Solution

We implemented a modular RAG system composed of four key components: (1) an NLP module for query parsing and entity extraction, (2) a vector search engine using FAISS, (3) a knowledge graph handler that queries DBpedia using SPARQL, and (4) an LLM module (ChatGPT) that synthesizes the results into a final response. This system enhances the factual reliability of generated answers while offering scalable performance.

Features/Functionality

  • End-to-end retrieval pipeline: from query to final LLM response
  • Wikipedia embeddings stored with FAISS for fast vector search
  • SPARQL-based querying of DBpedia for structured entity facts
  • Entity detection using natural language processing (spaCy)
  • Live performance logging of each module (e.g., NLP, KG, VS)
  • Frontend user interface built in React

Final Status

The project culminated in a fully functional minimum viable product that embeds and indexes the complete Wikipedia dataset to support scalable information retrieval.

A working end-to-end query pipeline was implemented, integrating the frontend user interface with the backend retrieval and generation components. User queries are processed through both vector search and knowledge graph retrieval, and the combined results are passed to a large language model to generate coherent, context-aware responses.
