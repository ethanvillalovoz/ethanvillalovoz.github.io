Abstract

This project implements a chatbot-style Retrieval-Augmented Generation (RAG) application for generating accurate, context-aware responses to user queries.

Built as a Washington State University senior design project for HackerEarth, the system combines vector search with knowledge-graph retrieval so responses can use both semantic similarity and structured entity relationships. The application processes Wikipedia-scale source material, stores embeddings with FAISS, retrieves structured facts from DBpedia through SPARQL, and synthesizes answers with an LLM-backed generation layer.

The final system follows a React/FastAPI client-server architecture and demonstrates an end-to-end pipeline from document ingestion and retrieval to grounded natural language output. Future work focuses on retrieval re-ranking, broader knowledge sources, and multimodal inputs.

System Architecture

We implemented a modular RAG system composed of four key components:

  • An NLP module for query parsing and entity extraction
  • A vector search engine using FAISS
  • A knowledge graph handler that queries DBpedia using SPARQL
  • An LLM module (ChatGPT) that synthesizes the results into a final response

This design improves factual grounding while keeping retrieval, graph lookup, and generation components independently testable.

Technical Features

  • End-to-end retrieval pipeline: query to final LLM response
  • Embeddings stored with FAISS for vector search
  • SPARQL-based querying (DBpedia) for structured entity facts
  • Entity detection using NLP (spaCy)
  • Live performance logging of modules (e.g., NLP, KG, VS)
  • Frontend user interface built in React

Final Status

The project culminated in a functional minimum viable product that embeds and indexes Wikipedia-scale source material to support scalable information retrieval.

A working end-to-end query pipeline was implemented, integrating the frontend user interface with the backend retrieval and generation components. User queries are processed through both vector search and knowledge graph retrieval, and the combined results are passed to a large language model to generate coherent, context-aware responses.

The resulting system met the project's technical objectives and was formally reviewed and approved by the client, HackerEarth, for final presentation.

Future Work

  • Expand knowledge sources beyond Wikipedia to include GitHub repositories, technical blogs, and curated news archives
  • Improve entity chunking strategies and retrieval re-ranking to increase search precision and response relevance
  • Fine-tune a lightweight LLM using domain-specific examples to improve factual grounding and generation quality
  • Extend the application to a multimodal interface supporting image, video, and audio-based user inputs
